Circular References

Cerial handles circular references in:

  • Lists, dicts, sets, tuples
  • Class instances
  • Any nested combination
a = []
a.append(a)  # self-referential list

obj = Node(1)
obj.next = obj  # self-referential object

# both serialize correctly

Not Supported

  • C extension objects without __reduce__ — no Python state to extract
  • Live network connections (mid-request) — state is external
  • Running threads/processes — cannot capture execution state
  • Memory views of external buffers — buffer not owned
  • ctypes pointers — raw memory addresses
